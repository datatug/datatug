# Contributing Guidelines for [DataTug](https://github.com/datatug/datatug)

We welcome contributions to DataTug!

## Feature requests

If you have an idea how DataTug can be improved:

1. First, check the existing
   [feature request](https://github.com/datatug/datatug/issues?q=is%3Aissue%20state%3Aopen%20label%3A%22feature%20request%22)
   – if possible avoid duplicates.

2. [Create a GitHub issue](https://github.com/datatug/datatug/issues/new) with the label `feature request`

## Bug reports

If you find a bug in DataTug:

1. First, check the existing
   [bug reports](https://github.com/datatug/datatug/issues?q=is%3Aissue%20state%3Aopen%20label%3A%22bug%22)
   – if possible avoid duplicates.

2. [Create a GitHub issue](https://github.com/datatug/datatug/issues/new) with the label `bug`

## If you want to contribute code

For a start check [issues](https://github.com/datatug/datatug/issues)
tagged with [`help wanted`](https://github.com/datatug/datatug/labels/help%20wanted)
and [`good first issue`](https://github.com/datatug/datatug/labels/good%20first%20issue).
